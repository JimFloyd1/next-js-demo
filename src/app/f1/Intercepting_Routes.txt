Intercepting routes allow you to intercept or stop the default routing behaviour to present an alternate view or component when navigating through the UI, while still preserving the intended route for senarios like page reloads.
This can be useful if you want to show a route while keeping the context of the current page
(.) to match segments on the same level
(..) to match segments on level above
(..)(..) to match segments two levels above
(...) to match segments from the root app directory
